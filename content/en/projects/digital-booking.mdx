## **Overview**

**Digital Booking** is a web application built as the capstone project for the **Certified Tech Developer** program (Digital House). The platform allows users to book accommodations—hotels, apartments, hostels, and more—from anywhere in the world at any time of the year, with an experience focused on clarity and usability.

This project brought together everything learned during the program, combining technical execution (frontend, backend, testing, database, and infrastructure) with soft skills such as collaboration, communication, and delivery discipline.

---

## **Goal**

The team’s goal was to consolidate the program’s learning outcomes by delivering a complete, functional end-to-end product, applying good engineering practices while taking UX/UI into account. Beyond building features, we focused on working like a real product team: planning, estimating, executing, and improving iteratively through feedback and retrospectives.

---

## **Team & delivery process**

The development team included **Arcila, Daniel; Borda Méndez, Ruth; Ferraro, María Eugenia; López, Daniel; Murillo, Juan; Rivera López, Jairo**. The project was executed using **Scrum**, with defined roles (Scrum Master, Product Owner, Principal Tech Lead, and tech leads per area) and a structure of **four 12-day sprints**.

We ran planning sessions, validation/estimation (Scrum Poker), dailies and weeklies, sprint reviews to demo progress, and retrospectives to refine our process. As the sprints progressed, we improved workload distribution, helped each other unblock issues, and strengthened CI/integration so we could deploy and validate in the cloud instead of only locally.

---

## **Technologies & tooling**

On the frontend, we used **JavaScript and React** to build the UI and the booking flows. The backend was built with **Java and Spring** to implement the domain logic and the required APIs. Persistence was handled with **MySQL** (with MySQL Workbench for administration and modeling).

For testing and validation, we used tools such as **Jest**, **Selenium**, and **Postman** to cover test flows and verify API behavior. Infrastructure work included **AWS** services like **EC2**, **S3**, and **RDS** to run, store, and operate the app in a production-like environment.

Project management relied on tools like **Trello**, **GitLab Issues**, **Metro Retro**, and **ClickUp**, along with a GitLab-based workflow for version control and tracking.
